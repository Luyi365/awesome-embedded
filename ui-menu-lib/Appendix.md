## 何为渲染？

参考：[渲染 - 维基百科，自由的百科全书 (wikipedia.org)](https://zh.wikipedia.org/zh-cn/%E6%B8%B2%E6%9F%93)

传统意义上的渲染指的是将代码变成五彩斑斓的页面过程。  
而现代渲染通常指的是一种创造性的行为，通过添加深度、细节和光影等元素来增加画面的真实感和丰富度。在嵌入式中，渲染可能将图形内容转换为可在嵌入式设备屏幕上显示的图像或视频的过程，这个过程涉及到照明、材质、纹理和视觉效果等多种技术和工具，目的是使内容看起来更加真实和生动。与 GUI 库的区别在于它属于 GUI 的一部分，专门做图形效果着色展示，需要大量的运算绘制。  

“渲染”这个概念在嵌入式端很抽象，通常在纯软件端，渲染指的是给通过给机器文本或代码的形式，以 **“生成最终的、给人看的图像”** ，比如浏览器渲染 HTML/CSS 成网页，游戏引擎渲染 3D 场景。  
而在嵌入式领域，“渲染”这个词被借用，但其核心含义变成了 **“将抽象的数据或状态，转换为驱动硬件的具体指令”** ，这比纯软件多了一道显示终端的适配。  
所以，当从软件领域来到嵌入式领域，听到“渲染”时，会觉得抽象是因为你期待的是「画图」，但实际上它干的是「翻译和驱动」的活儿。  

当然你也可以将其理解为绘制一些基本图案，例如点、圆、矩形等，但这里的绘制要经过两个层面，与内存一样，显示分为软件和硬件层面，软件指的是 GUI 绘制库，硬件指的是驱动屏幕的点亮。
